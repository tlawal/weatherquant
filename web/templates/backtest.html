How does this work?

1. We load every resolved market from your database (events where the winning bucket is known).

2. For each event, we replay the model's probability forecast + market prices that existed at the time.

3. We simulate trades using Kelly sizing, check for quick-flip exits, then resolve at the actual outcome.

4. Walk-forward mode splits data into rolling train/test windows and optimizes parameters on the training set before testing — this prevents overfitting.

Tip: If "Resolved Events" shows 0, click Enrich from Gamma to pull historical outcomes from the Polymarket API.

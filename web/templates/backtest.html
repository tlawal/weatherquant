Walk-Forward Backtester

Replay historical trading decisions against resolved Polymarket outcomes. Walk-forward optimization trains on rolling windows — no lookahead bias.

Replayable: Brier-only: Outcome-only:
How does this work?

1. We load every resolved market from your database (events where the winning bucket is known).

2. For each event, we replay the model's probability forecast + market prices that existed at the time.

3. We simulate trades using Kelly sizing, check for quick-flip exits, then resolve at the actual outcome.

4. Walk-forward mode splits data into rolling train/test windows and optimizes parameters on the training set before testing — this prevents overfitting.

Tip: The page can only simulate trades on events the bot tracked while they were open (full model_snapshots + market_snapshots history). Enrich from Gamma imports historical outcomes for reference, but it cannot synthesize the forecasts and orderbooks that weren't logged at the time. Replayable grows by ~1/city/day once the bot is armed.
